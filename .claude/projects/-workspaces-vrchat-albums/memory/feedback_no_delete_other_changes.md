---
name: 他プロセスの変更を絶対に消さない
description: 並列作業時に他のClaude Codeプロセスの変更を勝手に削除・リバート・上書きしてはならない。競合時はworktreeに切り替える。
type: feedback
---

他のプロセスや人間が行った変更を、自分の判断で削除・リバート・上書きしてはならない。

**Why:** 複数のClaude Codeプロセスで並行作業していたとき、あるプロセスが「余分な変更がありますね、消しておきます」と言って別プロセスの作業成果を削除してしまった。これにより作業が失われ、非常に良くない体験だった。

**How to apply:**
- セッション開始時に `git status` で未コミットの変更を確認し、自分のタスクと無関係な変更があれば worktree を切って作業開始する
- 作業中に自分が書いていない差分を発見したら、削除せず worktree に切り替える
- 判断がつかない場合は安全側に倒して worktree を使う
- この方針は全リポジトリ共通。ziku 経由で `.claude/rules/parallel-work.md` と hookify ルールが配布される
