# UI/UX デザインガイドライン

## 設計哲学: 引き算のデザイン

写真がコンテンツの主役。UIクロームは最小限。「何を追加しないか」を考える。

| 原則                 | 説明                                         |
| -------------------- | -------------------------------------------- |
| コンテンツファースト | 写真の視認性を最大化                         |
| 一貫性 > 独創性      | 既存パターンの踏襲が最善                     |
| 機能 > 装飾          | すべてのUI要素に「なぜ必要か」の理由が必要   |
| 静寂 > 賑やかさ      | アニメーションや色はアクションへの反応時のみ |

---

## 必須: 実装前プロセス

1. **既存パターン調査**: `src/components/ui/`, `src/v2/components/` で既存コンポーネントを検索。80%以上満たせるなら新規作成しない
2. **影響範囲確認**: ライト/ダークモード、ウィンドウサイズ変化への対応
3. **最小構成設計**: 「削除しても機能するか？」を各要素に問い、削除可能なら削除

---

## デザイントークン（直接指定禁止）

**色・サイズの直接指定（`bg-blue-500`, `text-4xl` 等）は禁止。** セマンティックトークンを使用。

| 用途           | トークン          | 参照先: `src/v2/constants/ui.ts`           |
| -------------- | ----------------- | ------------------------------------------ |
| テキスト       | `TEXT_COLOR.*`    | `text-foreground`, `text-muted-foreground` |
| 背景           | `SURFACE_COLOR.*` | `bg-background`, `bg-card`, `bg-muted`     |
| ステータス     | `STATUS_COLOR.*`  | `bg-success/10 text-success`               |
| ボーダー       | `BORDER.*`        | `border border-border`                     |
| タイポグラフィ | `TYPOGRAPHY.*`    | display/heading/body/caption               |
| スペーシング   | `SPACING.*`       | 8px グリッドベース                         |
| アイコンサイズ | `ICON_SIZE.*`     | xs(3)/sm(4)/md(5)/lg(8)                    |

---

## 禁止事項

| パターン                                | 代替                              |
| --------------------------------------- | --------------------------------- |
| `bg-blue-500` 等の直接色指定            | セマンティックトークン            |
| グラデーションテキスト                  | `text-foreground`                 |
| `rounded-3xl`                           | `rounded-md`（`--radius` ベース） |
| 常時アニメーション (`animate-pulse` 等) | 状態変化時のみ                    |
| カードの入れ子構造                      | フラットな構造                    |
| 大きなバナー/ヒーローセクション         | コンパクトなヘッダー              |
| ハードコードテキスト                    | `t()` 関数経由                    |
| 既存コンポーネントの再発明              | `src/components/ui/` の再利用     |

### アイコン使用基準

- **使用する**: ナビゲーション要素、アクションボタン、ステータスインジケーター
- **使用しない**: フォームラベルの装飾、テキストで十分な箇所

### アニメーション許可基準

- ✅ ユーザー操作フィードバック (`hover:scale-105`)、ローディングスピナー
- △ ダイアログの出現/消失（Radix UI 標準のみ）
- ✗ 常時アニメーション、ページ遷移

## コンポーネント設計

```text
既存コンポーネントで実現可能？ → Yes: 使用（variant追加も可）
組み合わせで実現可能？ → Yes: 組み合わせ
No → CVA + Radix UI パターンで新規作成
```

新規コンポーネントは CVA (class-variance-authority) パターン必須。`cn()` で結合。

### Glass morphism

- ✅ ダイアログ/モーダル (`glass-panel`)、Primary ボタン (`glass-button`)
- △ ヘッダー (`minimal-header`)、情報グルーピング用カード (`glass-card`)
- ✗ メインコンテンツ領域、入力フィールド

---

## UX 設計

- **情報量**: 今この瞬間に必要な情報だけ表示
- **フィードバック**: 100ms 以内の応答
- **ボタン配置**: プライマリ右寄せ、デストラクティブ左寄せ、間隔 `gap-2`
- **ローディング**: <1秒=表示なし、1-3秒=インラインスピナー、>3秒=メッセージ付き
- **空状態**: シンプルなテキスト + 必要時のみアクションボタン
- **ダーク/ライトモード**: セマンティックトークンで自動対応。`dark:` 手動指定は原則不要

---

## レイアウト

ヘッダー高さ `h-11` 固定。コンテンツ領域 `calc(100vh - 44px)`。モーダルは Radix UI Dialog。
